# Network Topology Visualizer

## Introduction

Network Topology Visualizer is a web-based tool for generating, uploading, and visualizing network topology data from Nmap traceroute results.

This project converts traceroute output into structured topology JSON data, then renders the topology as an interactive graph through a modern web interface.

## Content

### Topology Generation

The topology JSON file is derived from the text file generated by the `nmap` command's traceroute.

The traceroute text file contains the hop details from `127.0.0.1`, which is `localhost`, to each physical IP in the specified subnet. Each hop corresponds to a device name or an IP address.

The Python scripts process this traceroute output and convert it into a structured collection of network information containing nodes and links.

#### Traceroute Text File

Example traceroute text file:

```text
Starting Nmap 7.01 ( https://nmap.org ) at 2024-05-22 00:40 CST
Nmap scan report for 140.116.86.0
Host is up (0.00037s latency).

TRACEROUTE (using proto 1/icmp)
HOP RTT     ADDRESS
1   ... 4
5   0.30 ms 140.116.86.0

Nmap scan report for 140.116.86.1
Host is up (0.0031s latency).

TRACEROUTE (using proto 1/icmp)
HOP RTT      ADDRESS
1   0.25 ms  140.116.82.222
2   20.54 ms 140.116.243.89
3   10.93 ms 140.116.243.5
4   30.45 ms 140.116.243.226
5   0.56 ms  140.116.140.22
6   0.41 ms  140.116.86.1

...
```

### Topology JSON Format

The traceroute text file is processed into a topology JSON file.

The JSON file contains a `topo` object, which includes `nodes`, `portlists`, and `links`.

```json
{
  "topo": {
    "nodes": [
      ...
    ],
    "portlists": [],
    "links": [
      ...
    ]
  }
}
```

The structure of a node includes several parameters. The following example shows the format of a generated node:

```json
{
  "device_name": "140.116.86.100",
  "flag": 0,
  "gateway": "0.0.0.0",
  "id": "140.116.86.100",
  "ip": "140.116.86.100",
  "ip_on": true,
  "knmp_on": true,
  "mac": "",
  "netmask": "255.255.255.0",
  "snmp_on": true,
  "vlan": 1
}
```

Important node fields:

* `device_name`: The name of the device. If there is no device name, the IP address will be used.
* `id`: The unique identifier of each device. In this project, it is usually the IP address.
* `ip`: The IP address of the device.
* `gateway`: The gateway address.
* `netmask`: The network mask.
* `snmp_on`: The SNMP status of the device.
* `vlan`: The VLAN ID.

The structure of a link only lists the connection status between IPs.

```json
{
  "source": "127.0.0.1",
  "source_port_disp": "",
  "target": "140.116.86.100",
  "target_port_disp": ""
}
```

Important link fields:

* `source`: The source node ID.
* `target`: The target node ID.
* `source_port_disp`: Source port display field.
* `target_port_disp`: Target port display field.

### Web Visualizer

The web visualizer is built with React, TypeScript, Vite, and Cytoscape.js.

Users can upload a topology JSON file with the required `topo.nodes` and `topo.links` structure. After uploading the file, the web application renders the topology as an interactive network graph.

The visualizer supports basic topology inspection features, including node selection, node detail display, node search, and PNG export.

Main features:

* Upload topology JSON file
* Render interactive network graph
* Click nodes to inspect device information
* Search nodes by IP address or device name
* Export the visualized topology graph as a PNG image

## Usage

### 1. Install Nmap

```bash
sudo apt install nmap
```

---

### 2. Generate Traceroute Text File

Command format:

```bash
python src/nmap_traceroute.py <ip_mask> <txt_file_name>
```

Example:

```bash
python src/nmap_traceroute.py 192.168.44.* nmap_output.txt
```

---

### 3. Generate Topology JSON File

Command format:

```bash
python src/topo_generation.py <txt_file_name>
```

Example:

```bash
python src/topo_generation.py nmap_output.txt
```

---

### 4. Start Web Visualizer

Move into the web directory:

```bash
cd web
```

Install dependencies:

```bash
npm install
```

Start development server:

```bash
npm run dev
```

Open the local URL shown in the terminal.

Example:

```text
http://localhost:5173/
```

---

### 5. Web App Usage

1. Open the web application
2. Click `Choose JSON File`
3. Upload a topology JSON file
4. View the generated topology graph
5. Click nodes to inspect details
6. Search nodes by IP or device name
7. Click `Export PNG` to download the topology image
